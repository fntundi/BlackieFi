# BlackieFi

BlackieFi is now structured as a self-hosted stack instead of a Base44-published app.

## Architecture

- `src/`: existing React/Vite frontend, kept visually intact and rerouted through a local compatibility client
- `apps/api/`: TypeScript API service for auth, CRUD, reporting, automation, file ingest, and AI orchestration
- `services/llm-mcp/`: Python LLM integration service with provider-agnostic configuration and MCP-ready tool entrypoints
- `infra/k8s/`: Kubernetes manifests for PostgreSQL, API, web, LLM service, ingress, and recurring-job automation
- `infra/k3d/`: local k3d cluster template used by the Make-based bootstrap flow
- `Makefile`: Ubuntu-oriented local cluster bootstrap, image build, push, and deploy flow

## Local Web URL

- App: value of `APP_BASE_URL` from `env/.env.prod`

## Local Development

Frontend:

```bash
npm install
npm run dev:web
```

API:

```bash
npm --prefix apps/api install
npm run dev:api
```

LLM service:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r services/llm-mcp/requirements.txt
cd services/llm-mcp
uvicorn app.api:app --host 0.0.0.0 --port 8001
```

## Kubernetes Deployment

Ubuntu prerequisites:

- Docker
- `make`

Bootstrap tools:

```bash
make install-k3d
```

`k3d` is installed into `~/.local/bin`, and the `Makefile` prepends that location to `PATH` so the cluster targets can use it without requiring `sudo`.

Create the cluster, build images, push them to the local registry, apply manifests, and wait for rollout:

```bash
make deploy
```

Deployment hostname and base URL live in [env/.env.prod](C:/Users/FaithNtundi/Documents/Business/BlackieFi/BlackieFi/env/.env.prod). Update that file instead of editing manifests or the `Makefile`.

Useful overrides:

```bash
make deploy \
  POSTGRES_PASSWORD='postgres' \
  AUTOMATION_SHARED_KEY='replace-me' \
  LLM_PROVIDER_TYPE='ollama' \
  LLM_BASE_URL='http://host.k3d.internal:11434' \
  LLM_MODEL='llama3.1'
```

Other common commands:

```bash
make status
make logs-api
make logs-web
make logs-llm
make cluster-down
make clean
```

## AI Provider Configuration

Supported provider modes in the LLM service:

- `disabled`
- `ollama`
- `openai`
- `openai-compatible`
- `openrouter`
- `anthropic`
- `claude`

These are configured through Kubernetes secrets generated by the `Makefile`.
